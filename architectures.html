Image Classification Architectures

Dataset

I'll be using the CIFAR-10 dataset for all the architectures below. Since CIFAR-10 images are 32x32x3 (width, height, color channels), and many networks we'll be looking at were trained on imagenet cropped down to 224x224x3, the networks may not work as intended. So rather than scaling the 32x32x3 images up, I'll be implementing significantly scaled down versions of the networks while at the same time trying to preserve the main ideas unique to each architecture. For those unfamiliar with CIFAR-10, here's a quick summary. CIFAR-10 contains 10 classes: airplane, automobile, bird, cat, deer, dog, frog, horse, ship, truck Here are some random images from each of the classes. There are 60,000 images, with 6,000 images per class. We will use 50,000 for training/validation and the remaining 10,000 for testing. Of the 50,000 used for training and validation, we'll use a 80:20 split for training:validation.

GoogLeNet

Description

Tries to solve problem of overfitting with large/deep models, and high computational cost Uses dense blocks to create a sparse architecture Sparse structure is similar to how brain works (Hebbian principle - 'wire together, fire together'). Also computionally cheaper The input of the inception block is split into four parallel paths: The first three have 1x1, 3x3, and 5x5 kernel sizes, and the last is a max pooling layer (not sure why) A 1x1 conv layer is put in front of the 3x3 and 5x5 paths to reduce the number of channels (the whole sparsity thing, remember?) The output of the four pathways are then concatenated to form the output of the block Padding is added where necessary to make sure all inputs and outputs have the same width/height The input is first followed by a few conv layers. This is followed by 9 inception blocks. A global average pooling layer and a fully connected layer produces the output of the network.

Custom Implementation

We'll keep the inception blocks mostly the same, but strip out two of the three convolution/pooling layers at the beginning to prevent the image from shrinking down too much. The number of inception blocks will also be reduced from a total of nine to four to speed up computation. The original paper maintained two auxiliary outputs partway through the network that won't be included here. As more inception blocks are added, the width and height dimensions decrease after each pooling layer, but the depth channels increase. The actual values will be scaled down but we will follow the same pattern of decreasing width/height, and increasing channels as the data propagates deeper into the network. The local responsive normalization layers (which now have fallen out of favor) are replaced with the more standard batch normalization. See the pytoch implementation here.
